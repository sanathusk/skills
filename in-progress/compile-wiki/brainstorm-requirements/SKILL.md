## Refinement & Documentation Process

1. **Relentless Interviewing:**
   - Challenge the requirements against the existing domain model, system architecture, and code constraints.
   - Walk down each branch of the design tree, resolving dependencies between decisions one-by-one.
   - Ask questions one at a time, proposing a recommended answer for each question and waiting for feedback before proceeding.
   - If a question can be answered by exploring the codebase, do that first instead of asking.

2. **Immediate Documentation Updates:**
    Do not batch updates. As soon as any requirement, terminology, or technical detail is finalized or discovered, perform the updates inline according to the [Documentation Update Rules](../documentation-update-rules.md).
